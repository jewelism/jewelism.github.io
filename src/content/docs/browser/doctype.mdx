[웹표준](/browser/web-standard)에 대한 내용의 일부이다.

요즘 개발된 웹페이지 상단을 보면

```html
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
```

이런 태그를 본적있을 것이다.

저 태그를 DTD(document type definition)라고 한다.

태그를 삽입하지않아도 페이지가 작동하지만 중요한 이유가 있다.

기본적으로는 문서 형식 선언을 이용해서 어떤 웹 페이지가 모든 웹 브라우저에서 같은 레이아웃으로 제공되도록 할 수 있다.

### 표준모드(standards mode)와 쿽스모드(quirks mode)

브라우저는 선언된 doctype에 따라 렌더링할 모드를 선택하게 되는데 이 과정을 doctype sniffing 또는 doctype switching이라고 한다.

DOCTYPE 선언 내의 URL이 생략된 경우, 브라우저는 document를 quirks mode로 해석한다.

위에서 말했던것처럼, 표준모드라는 것은 모든 웹 브라우저에서 같은 레이아웃으로 제공될수있도록 함이다.

DTD을 제대로 선언하지않으면, quirks mode로 렌더링하는데 브라우저마다 레이아웃이 달라질 수 있다.

SEO에도 중요하다. 웹사이트의 점수가 낮으면, 나의 웹사이트 순위는 점점 내려간다.
